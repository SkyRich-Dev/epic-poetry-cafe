Project: Epic Poetry Cafe – Simple Operations, Costing, Waste, Trial/R&D, and Analytics System
Build a production-ready full-stack web application for Epic Poetry Cafe using:
Frontend: React JS with Vite, MUI, CSS
Backend: Python Django + Django REST Framework
Database: PostgreSQL
Auth: JWT authentication
Architecture: Clean modular architecture, scalable, secure, configurable
UI Goal: Very simple daily-use interface with powerful analytics
Primary Users: Owner, Manager, Store/Purchase User, Kitchen/Bar User, Accounts/Admin
Design Goal: Minimal data entry, strong calculations, strong analytics, mobile/tablet friendly
This system is for a single cafe outlet initially, but the architecture must be ready for multi-outlet expansion later without major redesign.

2.3 No hard-coded business values

2.4 Auditability

25. CALCULATION LOGIC DETAILS
Implement these formulas carefully.
25.1 Ingredient line cost in recipe
ingredient line cost = effective ingredient cost per recipe UOM × net quantity required
25.2 Net quantity required
net quantity required = quantity required + wastage quantity
or
net quantity required = quantity required × (1 + wastage_percent/100)
25.3 Item production cost
sum of all recipe line costs
25.4 Item margin
selling price - production cost
25.5 Margin %
(item margin / selling price) × 100
25.6 Actual stock consumption
opening + inward - closing - waste - trial
25.7 Theoretical consumption
sum of all sold quantities × recipe quantities
25.8 Variance
actual consumption - theoretical consumption
25.9 Daily waste %
daily waste / daily sales × 100
25.10 Daily allocated fixed cost
monthly fixed amount / number of days in month
25.11 Daily estimated operating profit
sales - direct material cost - waste cost - trial cost - daily allocated expenses - daily allocated utilities
